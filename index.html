Realtime Systems

Realtime Notification Center

Multi-tab notification system with distributed state synchronization for a B2B communication platform.

NestJS PostgreSQL Pusher React TypeScript Redis

Overview

Designed and implemented a realtime notification infrastructure for a healthcare communication platform supporting appointment lifecycle events. The system delivers consistent, deduplicated notifications across multiple browser tabs and sessions, with persistent unread state synchronization.

Architecture

  • Pusher channels for realtime event broadcasting to connected clients
  • NestJS event emitters for server-side lifecycle triggers
  • PostgreSQL for persistent notification and read-state storage
  • Redis for cross-instance event coordination
  • React + custom hooks for client-side subscription management

Engineering Challenges

  • Preventing duplicate browser notifications when multiple tabs are open for the same session
  • Synchronizing read/unread state across tabs in realtime without a SharedWorker
  • Handling websocket reconnection gracefully without missing events during the gap
  • Token refresh without interrupting active Pusher subscriptions
  • Ensuring notification ordering consistency under high-frequency event bursts

Key Decision

Used per-tab Pusher connections with a leader-election pattern (via localStorage events) to designate which tab owns browser notification dispatch — eliminating duplicates without the complexity of a SharedWorker. Simpler to debug and zero cross-origin issues.

Impact

Eliminated missed appointment events for support agents, ensuring reliable notification delivery regardless of how many tabs or sessions were open simultaneously.

Communication Infrastructure

Twilio Communication Platform

Large-scale real-time communications infrastructure across voice, video, messaging, and intelligent task routing.

Twilio TaskRouter NestJS Inngest AWS Lambda TypeScript

Overview

Designed and operated communication infrastructure for a global B2B platform, handling interaction workflows across voice, video, messaging, and intelligent task routing. Built event-driven orchestration with Inngest and AWS Lambda for communication lifecycle automation at scale.

Architecture

  • Twilio TaskRouter for skills-based task assignment across worker pools
  • Twilio Voice, Video, and Messaging APIs for channel handling
  • Inngest for reliable background job orchestration with built-in retries
  • AWS Lambda for stateless communication lifecycle event processing
  • Custom useTaskRouter React hook for frontend worker state management

Engineering Challenges

  • Managing TaskRouter worker lifecycle including token expiry and refresh without dropping reservations
  • Handling reservation state transitions across network interruptions
  • IVR fallback routing when primary queues are unavailable
  • Coordinating async communication events across distributed Lambda invocations
  • Maintaining worker availability state reliably under concurrent task assignments

Key Decision

Built a custom useTaskRouter hook encapsulating the full worker lifecycle — registration, token refresh, reservation handling, and disconnection recovery — rather than spreading this logic across components. This made the TaskRouter integration testable in isolation and reusable across different UI surfaces.

Impact

Enabled reliable task routing across multiple communication channels for a global workforce, with resilient token management preventing dropped reservations during extended sessions.

Cloud Infrastructure

AWS Cloud Migration & Infrastructure

Led Azure Blob → AWS S3 migration with zero downtime and LocalStack-powered integration test coverage.

AWS S3 AWS ECS AWS Lambda LocalStack Docker NestJS

Overview

Led the infrastructure migration of a production file storage system from Azure Blob Storage to AWS S3, implementing a drop-in provider abstraction that allowed seamless cutover without application-layer changes. Hardened ECS deployments and automated task definitions to improve release stability across environments.

Architecture

  • Storage provider abstraction layer (interface-based) in NestJS
  • AWS S3 as the target with IAM role-based access control
  • LocalStack for local and CI integration testing of S3 operations
  • ECS task definitions managed via infrastructure-as-code patterns
  • Zero-downtime migration strategy using dual-write transitional phase

Engineering Challenges

  • Ensuring zero downtime during cutover without a maintenance window
  • Writing integration tests that run identically in local dev, CI, and staging
  • Hardening ECS task definitions for predictable rolling deployments
  • Abstracting provider-specific S3 and Azure APIs behind a unified interface

Key Decision

Implemented a NestJS storage provider abstraction injected via dependency injection rather than calling the S3 SDK directly throughout the codebase. The migration became a single injection swap instead of hundreds of call-site changes, and tests stay clean by swapping in a LocalStack provider at test time.

Impact

Zero-downtime migration with full integration test coverage across local, CI, and staging environments. Improved deployment release stability and reduced cloud infrastructure costs.

Frontend Engineering

High-Performance Virtualized UI System

Scalable frontend components handling large datasets with virtualization, server-driven filtering, and keyboard navigation.

React TypeScript React Query NestJS PostgreSQL

Overview

Built a suite of high-performance virtualized React components — multi-select dropdowns, server-driven data tables, and calendar views — capable of handling large datasets without rendering bottlenecks. Integrated with server-side filtering and real-time analytics dashboards via React Query.

Architecture

  • Virtual list rendering — only visible rows mounted in the DOM at any time
  • Server-driven filtering architecture with filter state in URL params, not component state
  • React Query for data fetching, caching, and background refetch
  • Custom keyboard navigation system for virtualized multi-select
  • NestJS reusable server-side filter layer supporting multiple client views

Engineering Challenges

  • Maintaining smooth 60fps scrolling in tables with 10,000+ rows
  • Keyboard navigation in a virtualized list where items are not always mounted
  • Synchronizing server-side filter state with optimistic UI updates
  • Building a calendar component that handles dense event data without layout thrashing

Key Decision

Kept filter state server-driven (URL params + React Query) rather than local component state. This made all views bookmarkable, shareable, and independently refreshable — a critical requirement for support agent workflows that needed to resume sessions mid-filter.

Impact

Eliminated rendering jank for large data tables and multi-select interactions, significantly improving support agent productivity on high-volume daily workflows.
